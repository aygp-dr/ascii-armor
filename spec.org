* Purpose and Scope

ascii-armor validates the toolchain and encoding patterns for
binary-in-text encoding across the following lineage:

- uuencode / uudecode (BSD 4.0, 1980)
- GNU shar / unshar (shell archives)
- BinHex 4.0 / MacBinary (Mac resource-fork preservation)
- XBM / XPM (X11 C-source image formats)
- X-Face / compface (email header bitmaps)
- MIME base64 (RFC 2045)
- RFC 2397 data: URI scheme
- Ascii85 / Base85 (PostScript/PDF/Git)
- Git binary patches (~--binary~)
- Org-mode named-block + babel decode pattern

Primary delivery: a single ~spec.org~ in which every code block is
tangleable, every claim is executable, and every tool dependency is
declared.  The spec is the test suite.

* Foundational Axiom

#+begin_quote
Every binary-in-text encoding is a solution to the same constraint:
a binary artifact must survive a channel that only tolerates text.
The encoding, the wrapper, and the decode step are structurally identical
across all instances; only the channel and the alphabet differ.
#+end_quote

The purpose of this repo is to make that structural identity concrete
and executable on FreeBSD.

* Anti-Goals

Do not build any of the following:

- A new encoding format (not invented here)
- A general-purpose compression library
- A web application or REST API
- A production file transfer tool
- Anything that requires a network connection to pass tests
- A reimplementation of ~base64(1)~, ~uuencode(1)~, or ~xxd(1)~

* Design Principles

1. *Spec is the test suite.* Every ~#+begin_src~ block must be
   executable via ~C-c C-c~ or ~org-babel-execute-buffer~.  If it
   cannot be executed, it is documentation only and must be marked
   ~:eval no~.

2. *Round-trip is the acceptance criterion.* For each encoding,
   the test is: ~encode(decode(encode(x))) == encode(x)~.  Partial
   coverage (encode only, no decode) is not sufficient.

3. *Tool detection before tool use.* Every shell block that calls
   an external tool must first verify it is on PATH using a
   prerequisite check block.  Missing tools are surfaced as
   structured errors, not silent failures.

4. *FreeBSD first.* All shell commands must run on FreeBSD 14.x.
   GNU coreutils variants are explicitly called out where they differ
   from BSD equivalents.

5. *Elisp for org-native operations.* Display, overlay, and inline
   image operations use ~emacs-lisp~ blocks.  Shell blocks handle
   encoding; elisp blocks handle presentation.

6. *No binary blobs in git.* Test fixtures are generated by the
   spec itself, not committed.  The spec is self-bootstrapping.

** Tool availability check

#+name: prereq-check
#+begin_src bash :tangle bin/check-prereqs.sh :shebang "#!/bin/sh"
#!/bin/sh
# ascii-armor prerequisite check
# Exit 0: all required tools present
# Exit 1: degraded (optional tools missing)
# Exit 2: broken (required tools missing)

REQUIRED="uuencode uudecode b64encode xxd git"
OPTIONAL="shar unshar compface uncompface convert ffmpeg"
PORTS="sharutils:shar graphics/compface:compface ImageMagick7:convert"

missing_required=0
missing_optional=0

check() {
  local tool="$1" req="$2"
  if command -v "$tool" >/dev/null 2>&1; then
    printf '{"tool":"%s","status":"ok","path":"%s"}\n' \
      "$tool" "$(command -v "$tool")"
  else
    printf '{"tool":"%s","status":"missing","required":%s}\n' \
      "$tool" "$req"
    [ "$req" = "true" ] && missing_required=$((missing_required+1)) \
                        || missing_optional=$((missing_optional+1))
  fi
}

echo '{"prereqs":['
for t in $REQUIRED; do check "$t" true;  done
for t in $OPTIONAL; do check "$t" false; done
echo ']}'

[ $missing_required -gt 0 ] && exit 2
[ $missing_optional -gt 0 ] && exit 1
exit 0
#+end_src

** Decode round-trip

#+name: uuencode-roundtrip
#+begin_src bash :tangle bin/test-uuencode.sh :shebang "#!/bin/sh" :results output
#!/bin/sh
set -e
INPUT=tests/fixtures/cat.txt
ENCODED=$(uuencode "$INPUT" cat.txt)
DECODED=$(echo "$ENCODED" | uudecode -o /dev/stdout 2>/dev/null \
          || echo "$ENCODED" | uudecode -p)

ORIG=$(cat "$INPUT")
if [ "$DECODED" = "$ORIG" ]; then
  echo "PASS uuencode round-trip: '$ORIG' → encoded → '$DECODED'"
else
  echo "FAIL uuencode round-trip: got '$DECODED' expected '$ORIG'"
  exit 1
fi

# Binary round-trip
uuencode tests/fixtures/binary.bin binary.bin > /tmp/binary.uu
uudecode -o /tmp/binary.out /tmp/binary.uu 2>/dev/null \
  || uudecode -p /tmp/binary.uu > /tmp/binary.out
if cmp -s tests/fixtures/binary.bin /tmp/binary.out; then
  echo "PASS uuencode binary round-trip (32 bytes)"
else
  echo "FAIL uuencode binary round-trip"
  exit 1
fi

** Python Ascii85 round-trip

#+begin_src python :tangle bin/test-ascii85.py :results output
#!/usr/bin/env python3
"""Ascii85 round-trip validation (Python stdlib)."""
import base64, sys

with open('tests/fixtures/binary.bin', 'rb') as f:
    data = f.read()

# Python uses b85encode which is ZeroMQ Z85 variant
# For Adobe Ascii85 we use the a85encode with adobe=True
encoded_a85 = base64.a85encode(data, adobe=True)
decoded_a85  = base64.a85decode(encoded_a85, adobe=True)

assert data == decoded_a85, "FAIL Ascii85 round-trip"
raw_len = len(data)
enc_len = len(encoded_a85)
overhead = (enc_len - raw_len) / raw_len * 100
print(f"PASS Ascii85 (Adobe) round-trip: raw={raw_len}B enc={enc_len}B overhead={overhead:.1f}%")
print(f"     encoded prefix: {encoded_a85[:40]}...")

# Compare overhead: base64 vs ascii85
encoded_b64 = base64.b64encode(data)
b64_oh = (len(encoded_b64) - raw_len) / raw_len * 100
print(f"     base64 overhead:  {b64_oh:.1f}%  (Ascii85 saves {b64_oh-overhead:.1f}%)")

* Encoding: org-mode named-block pattern (shar analogue)
:PROPERTIES:
:CPRR: C-010

This is the pattern used in ~mordhau-inline.org~: base64 payload in
a named ~#+begin_src text~ block, decoded by a babel eval block to

** Encode fixture to named block

#+begin_src emacs-lisp :results output
(let* ((png-file "tests/fixtures/red1x1.png")
       (data (with-temp-buffer
               (set-buffer-multibyte nil)
               (insert-file-contents-literally png-file)
               (buffer-string)))
       (b64  (base64-encode-string data))
       ;; Wrap at 76 chars for org block readability
       (wrapped (with-temp-buffer
                  (insert b64)
                  (goto-char (point-min))
                  (while (< (point) (point-max))
                    (forward-char 76)
                    (unless (eobp) (insert "\n")))
                  (buffer-string))))

#+name: overhead-comparison
#+begin_src python :tangle bin/overhead-comparison.py :results output
#!/usr/bin/env python3
"""Compare encoding overhead across all formats."""
import base64, subprocess, sys, os

def run(cmd):
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True)
    return r.stdout.strip() if r.returncode == 0 else None

def overhead(orig_len, enc_len):
    return (enc_len - orig_len) / orig_len * 100

with open('tests/fixtures/binary.bin', 'rb') as f:
    data = f.read()

orig = len(data)

results = []

# base64
b64 = base64.b64encode(data)
results.append(('MIME base64',      orig, len(b64),              overhead(orig, len(b64))))

# Ascii85 (Adobe)
a85 = base64.a85encode(data, adobe=True)
results.append(('Ascii85/PDF',      orig, len(a85),              overhead(orig, len(a85))))

# uuencode (approximate: each 3 bytes → 4 + line overhead)
uu_est = (orig + 2) // 3 * 4 + (orig // 45 + 1) * 2  # 60-char lines + headers
results.append(('uuencode (est)',   orig, uu_est,                overhead(orig, uu_est)))

# xxd hex
xxd_est = orig * 5  # "0xNN, " per byte roughly
results.append(('xxd -i / XBM hex',orig, xxd_est,               overhead(orig, xxd_est)))

# XPM chars_per_pixel=1 for ≤94 colors: 1 char/pixel (no expansion for image data itself)
xpm_est = orig * 8  # 1 bit/pixel → 1 char represents 1 bit; ~8:1 vs binary pixel
results.append(('XPM (1bpp est)',   orig, xpm_est,               overhead(orig, xpm_est)))

print(f"{'Format':<20} {'Raw':>6} {'Encoded':>8} {'Overhead':>10}")
print('-' * 48)
for name, raw, enc, oh in sorted(results, key=lambda x: x[3]):
    print(f"{name:<20} {raw:>6} {enc:>8} {oh:>9.1f}%")
#+end_src

# ascii-armor — Binary-in-Text Encoding Validation Suite

## Your Role

You are a coding agent.  Your job is to implement the validation
suite described in spec.org.  You produce working shell scripts,
Python scripts, Emacs Lisp, and org-mode babel blocks.  You do not
re-specify the system unless you find a direct contradiction in the
spec — in that case, surface it as a CPRR conjecture before proceeding.

## Confirmation Gate

Before writing any code, output a one-paragraph summary of what you
understand ascii-armor to be, who its primary user is, and what its
primary output is.  Do not proceed until this summary is accurate.

## What You Are Building

- A literate org-mode spec where every encoding claim is an executable
  round-trip test (encode → decode → compare to original)
- Shell/Python/Elisp test scripts tangled from spec.org, runnable on
  FreeBSD 14.x without network access
- An overhead comparison table covering uuencode, base64, Ascii85,
  xxd/XBM, and XPM

## Foundational Axiom

> Every binary-in-text encoding solves the same constraint: binary
> data must survive a text-only channel.  The HEADER/PAYLOAD/FOOTER
> structure and the ~33% size tax are invariant; only the alphabet
> and channel differ.

This is structural, not preferential.  Any new format that emerges
from this spec should be evaluated against this axiom first.

## Explicit Anti-Goals

Do not build any of the following:

- A new binary-to-text encoding format
- A web server or REST API
- A general-purpose compression library
- A reimplementation of base64(1), uuencode(1), or xxd(1)
- Any code that requires network access to pass tests

## Key Design Decisions

- spec.org is the source of truth; tangled scripts are derived artifacts
- Round-trip (encode→decode→compare) is the only acceptance criterion
- Tool detection (prereq check) precedes every tool invocation
- FreeBSD 14.x base system first; GNU/Linux variants called out explicitly
- Emacs Lisp handles org display; shell/Python handle encoding ops

## FreeBSD Constraint

All shell blocks must run on FreeBSD 14.x.  Note:
- `b64encode`/`b64decode` are in base (not GNU coreutils)
- `uuencode` is in base but differs from GNU sharutils variant
- `xxd` is in base
- `shar`/`compface` are ports (optional)
- `sha256` is `sha256(1)` not `sha256sum(1)`

## Build Order

Follow step order in spec.org §.  Do not skip layers.
Each step must have a passing round-trip test before proceeding.

**Failure handler**: If a round-trip test fails, stop.  Document what
failed in a #+RESULTS: block.  Surface as CPRR refutation.
Do not proceed to next encoding.

1. Prerequisites check (bin/check-prereqs.sh)
2. Test fixture generation (tests/gen-fixtures.sh)
3. uuencode/uudecode round-trip
4. b64encode/b64decode round-trip
5. xxd round-trip
6. XBM structure validation
7. XPM structure validation
8. MIME multipart generation + extraction
9. RFC 2397 data: URI generation
10. Ascii85 round-trip (Python)
11. Git binary patch generation + apply
12. Org named-block decode pattern (elisp)
13. X-Face (if compface available)
14. GNU shar (if available)
15. Overhead comparison table

## Open Conjectures (test these, do not assume)

- C-001: All required tools are present in FreeBSD 14 base
- C-002: Fixture generation is deterministic and self-contained
- C-003: uuencode overhead is ~37% for binary payloads
- C-004: b64encode overhead is ~33% for binary payloads
- C-005: xxd -i produces valid C array syntax parseable by GCC
- C-006: XBM byte count = ceil(width/8) * height exactly
- C-007: MIME multipart/related extraction preserves PNG magic bytes
- C-008: data: URI round-trip preserves binary exactly
- C-009: Ascii85 overhead is ~25% (better than base64)
- C-010: Org named-block babel decode is exact to source file
- C-011: compface is not in FreeBSD ports and must be built from source
- C-012: GNU shar differs from base uuencode in header format
- C-013: Ascii85 saves ~8pp overhead vs base64 for random binary data
- C-014: Elisp display helpers work without external dependencies

## Acceptance: End-to-End Test

Given the 32-byte binary fixture `tests/fixtures/binary.bin`:

- `make test` exits 0 on a stock FreeBSD 14 system with `git` installed
- All required encoding round-trips produce identical output to input
- The overhead comparison table shows Ascii85 < base64 < uuencode < xxd
- Org babel execution of spec.org completes without errors
